プロシージャル・依存グラフ型

頂点、属性、値の非破壊的関係性

「あらゆる属性をあらゆる属性に繋ぐ（Anything to anything）」設計。変形の計算は、マテリアルの貼り付けやピクセル化（ラスタライズ）よりも前に行われるため、無限に変形してもボケや劣化が全く起きません。また、キーフレームを一切打たずに、オシレーターやノイズの数値データだけを渡して駆動させることが可能です。

// 非破壊ベクターグラフ
Noise -> [Value2] -> Scale (ベクター計算)
Path -> Bend Deformer -> [Shape頂点再生成]
└ (最後にラスタライズ) -> Glow Filter

システム構造が生み出す決定的な表現力の境界

1. インデックス・コンテクスト (個別制御)

Cavalryでは1万個の複製を作った場合、各複製が「インデックス（配列番号）」を自覚しています。 NoiseやRandomモジュールをクローナー（Duplicator）に接続すると、自動的に「インデックスに応じた位相差」を生成し、それぞれ異なるランダム値が渡されます。 AviUtlでこれをやるには、複製後にLuaスクリプトで obj.index を使って自ら演算ループのコードを書く必要があります。

2. Scene Referencing (協調アセット参照)

Cavalryでは、あるプロジェクト（.cvファイル）を別のプロジェクトの中にアセットとしてインポートして参照することができます。 元ファイルを編集すれば、参照先にもリアルタイムに更新が自動伝播するため、チーム協調設計が極めて容易です。 AviUtlのプロジェクト（.aup）は独立して完結しており、一度動画に書き出して再インポートするか、手動でオブジェクト群をタイムラインへ力技でコピペするしかありません。
